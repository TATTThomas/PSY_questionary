各API說明

general users

無路由

  • get -> “./” ,no parameter
    • 渲染至EW的page1(首頁)

“./EW”

  • get -> “./EW” ,no parameter
    • 渲染至EW的page1(首頁)
  • post -> “./EW/login” ,parameters: ID:帳號,password:密碼
    • 比對帳號密碼,不配對則渲染至錯誤頁
    • 若為第一次進入系統, 攜帶帳號密碼渲染至page2(契約書頁) {ID,password}
    • 若非第一次進入系統, 攜帶帳號密碼以及該ID於personal_data中的個人資料渲染至page4(個人資料填寫頁) {ID,password,data}
  • post -> “./EW/jumpBoard” ,parameters: ID:帳號,password:密碼,goal:目標頁(ex:page1,page2,…)
    • 若goal 不屬於 => page3 page4 page5 則渲染至錯誤頁
    • 比對帳號密碼,不配對則渲染至錯誤頁
    • 配對則攜帶帳號密碼渲染至goal所描述的頁 {ID,password,data:NA}
  • post(AJAX) -> “./EW/changePassword” ,parameters: ID:帳號,password:密碼,new_password:新密碼
    • 比對帳號密碼,不配對則回傳result:error
    • 配對則修改personal_information的password
    • 回傳result:success
  • post(AJAX) -> “./EW/updateData” ,parameters: ID:帳號,password:密碼,personal_data中除ID外全參數
    • 比對帳號密碼,不配對則回傳result:error
    • 配對則update,該ID於personal_data中除了ID所有參數
    • update成功回傳result:success

“./GQ”

“./GQ/EQ”
  • post-> “./GQ/EQ”, parameters: ID:帳號,password:密碼
    • 比對帳號密碼,不配對則渲染至錯誤頁
    • 渲染至EQ(GQ.EQ)選擇問卷頁 攜帶ID,password
  • post-> “./GQ/EQ/:which”, parameters: ID:帳號,password:密碼
    • 若which不屬於A~M則渲染至錯誤頁
    • 比對帳號密碼,不配對則渲染至錯誤頁
    • 渲染至which問卷頁 攜帶ID,password
  • post-> “./GQ/EQ/PR” , parameters: ID:帳號,password:密碼
    • 比對帳號密碼,不配對則渲染至錯誤頁
    • 渲染至PR(GQ.PR)頁 攜帶ID,password
“./GQ/PR”
  • post(AJAX)-> “./GQ/PR/GetPersonalData”, parameters: ID:帳號,password:密碼
    • 比對帳號密碼,不配對則回傳result:error, data:NA
    • 配對則回傳在GQ_personal中的ID符合參數ID的document(json),result:success, data:json
  • post(AJAX)-> “./GQ/PR/GetCriticalData”, mode:年齡段
    • mode參數在許可列表 ,否則回傳result:error, data:NA
    • 回傳data.critical_value中mode符合傳入參數的document,屏蔽mode,_id回傳(json)
      result:success, data:json 。
“./GQ/SQ”
  • post(AJAX)-> “./GQ/SQ/saveData” parameters: ID:帳號,password:密碼,one:one的資料(string),group:group的資料(string),type:A~M
    • 比對帳號密碼,不配對則回傳result:error
    • 根據type選擇模式A~M, 模式不存在result:error
    • 拆解one,group,寫入時間 存入DB
    • 存入成功result:success, 失敗result:error
    • 各問卷傳遞格式詳見周進度

manage users

無路由

  • get-> “./”
    • 渲染至MDB的page1(登入頁)
  • post-> “./login” parameters: ID:帳號,password:密碼
    • 比對帳號密碼正確
    • 渲染至MDB的page2(系統選擇頁)
  • post-> “./base” parameters: ID:帳號,password:密碼
    • 比對帳號密碼正確
    • 渲染至MDB的page3(基本功能頁)
  • post-> “./GQ” parameters: ID:帳號,password:密碼
    • 比對帳號密碼正確
    • 渲染至MDB的page4(GQ功能頁)

“./Create”

  • post(AJAX)-> “./Create/newUser” parameters: ID:帳號,password:密碼,tag:一個英文字母用作帳號標記
    • 比對帳號密碼正確,有誤回傳result:error,ID:NA,password:NA
    • 抓取lock.users.usersCount 的count且+1 (findandupdate)
    • tag + _ + 取得值,就是新ID
    • 隨機生成亂碼為密碼
    • EW.personal_information 插入{ID,password,first:True}的document
    • 都成功就回傳result:success,ID:new ID,password:random password

“./Get”

  • post(AJAX)-> “./Get/table” parameters: ID:帳號,password:密碼,DB:目標資料庫,collection:目標目錄,fillterKey:XX,fillterValue:XX,type:1/0(要不要用fillter?)
    • 比對帳號密碼正確,有誤回傳result:error,data:NA
    • DB.collection.find({fillterKey:fillterValue})
    • 把收到的jsonList or json回傳
    • 正確回傳 result:success,data: json

“./Update”

  • post(AJAX)-> “./Update/table” parameters: ID:帳號,password:密碼,data:表格資料csv string,where:哪一個表格
    • 比對帳號密碼正確,有誤回傳result:error
    • where的collection在許可列表中
    • 該collection清空
    • 該csv string一行一個JSON存入
    • 正確回傳 result:success
  • post(AJAX)-> “./Update/criticalNumber” parameters: ID:帳號,password:密碼,data:表格資料csv string,where:哪一個表格
    • 比對帳號密碼正確,有誤回傳result:error
    • where的collection在許可列表中(與上方功能差異僅在許可列表不同)
    • 該collection清空
    • 該csv string一行一個JSON存入
    • 正確回傳 result:success
