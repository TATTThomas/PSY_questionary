新增API “./GQ/PR/savePR” (post AJAX)

功能

存入個人資料至個人報告子系統, 要存入的資料都來源於該頁的group資料裡

參數

ID = 帳號
password = 密碼
type = 問卷名稱 ex:A,B,C,…
data = 數據,用_相隔, 若該問卷要傳回3個參數, 及存入 : p1_p2_p3

個問卷data填入值

新版規格書:https://drive.google.com/open?id=1IjCvdjVo0yQ8t5o0fxx8mdfnULov57hk

‘B’: [‘A2’, ‘B2’, ‘C2’],由左至右分別是B.group=>Acc,RT,SSRT
‘C’: [‘A3’, ‘C3’],C.group=>Acc,平均一致性移動比率

AJAX函數回傳作法完整化

說明

有兩種回傳模式
1.submit=> 傳輸失敗會跳警告頁,此類API由後端處理
2.AJAX =>

  • 傳輸成功回傳{result:“success”}
  • 傳輸失敗回傳{result: 失敗原因(string) }
然而以上兩種對網頁來說都屬於成功,所以以下提供建議處理作法

範例代碼

$.post("/GQ/SQ/saveData", {
                    ID: "<%=ID%>",
                    password: "<%=password%>",
                    one: localStorage.getItem("one"),
                    group: "NA",
                    type: 'L',
                },
                    function (data, textStatus, jqXHR) {
                        if (textStatus == "success") {
                            if (data.result == "success") {
                                localStorage.clear();
                                alert("上次未上傳資料上傳成功");
                            }
                            else {
                                alert("資料上傳失敗,失敗原因 : " + data.result);
                            }
                        }
                        else
                            alert("與伺服器斷訊")
                    },
                    "json"
                );

新增回合斷點與儲存功能

具回合制

有幾份問卷因時辰過長,所以分成多回合,在回合與回合中間插入說明頁(含下一回合按鈕供點擊,不含離開按鈕或其他任意按鈕),說明進入第幾回合(若下回合規則有異則輔已說明文字),且儲存該進度資料,若之後進入該頁(且有未寄出資料),將繼承之前進度繼續運行(若此功能過於複雜,可改為直接寄出,後面全補NA)

各需該功能問卷細說

C,D

分三回合,中間須出現2次說明頁(需文字內容[共三回合,接下來是第X回合], 按鈕[進入下一頁])
練習模式無關此功能

H

分三回合,中間須出現2次說明頁(需文字內容告知回合以及該回合測試內容(內容請至點這), 按鈕[進入下一頁])
練習模式題數減少,亦須說明頁,但無須紀錄資料
